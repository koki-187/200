## 売側ユーザーの役割

### あなたの役割
売側ユーザーは、**土地の売却を希望する側**として、買側からの問い合わせに回答します。

**主な業務**:
- ✅ 買側からの案件レビュー
- ✅ 物件情報の確認・追加入力
- ✅ 必要書類のアップロード
- ✅ 買側への回答作成
- ✅ 質問への対応
- ✅ 条件交渉・成約判断

**権限**:
- ✅ 自社に割り当てられた案件の閲覧・編集
- ✅ 案件へのメッセージ送信
- ✅ ファイルのアップロード・ダウンロード
- ❌ 他社の案件は閲覧不可
- ❌ 新規案件の登録は不可（買側のみ）
- ❌ ユーザー管理・システム設定は不可

## ログイン方法

### ステップ1: ログインページにアクセス
```
URL: https://real-estate-200units-v2.pages.dev/
```

### ステップ2: 売側アカウントでログイン
- **メールアドレス**: 管理者から提供されたメールアドレス
- **パスワード**: 初回ログイン時に設定したパスワード
- **ロール**: AGENT（売側として案件対応）

### ステップ3: ダッシュボードへ自動遷移
ログイン成功後、自動的にダッシュボードに遷移します。

### セキュリティのベストプラクティス
- 強力なパスワードを使用（12文字以上、英数字記号混在）
- パスワードを他人と共有しない
- 定期的にパスワードを変更（3ヶ月ごと推奨）
- 共用PCでは必ずログアウト

## ダッシュボード

### 概要
ダッシュボードでは、あなたに割り当てられた案件の状況を一覧で確認できます。

### KPIカード

#### 1. 総案件数
- 表示内容: あなたに割り当てられた全案件の総数
- 用途: 自社の対応案件ボリュームを把握

#### 2. 進行中
- 表示内容: ステータスが「新規」「レビュー中」の案件数
- 用途: 回答が必要な案件を把握
- アクション: **重要**: この数値が多い場合、速やかにレビューが必要

#### 3. 回答済み
- 表示内容: ステータスが「回答済み」の案件数
- 用途: 買側の確認待ち案件を把握
- アクション: 買側からの追加質問に備える

#### 4. 完了
- 表示内容: ステータスが「終了」の案件数
- 用途: 成約または不成立で終了した案件数
- アクション: 成約案件は契約手続きへ進む

### 最近の案件
- 表示件数: 最新5件
- 表示情報: 案件名、ステータス、所在地、作成日
- アクション: クリックで案件詳細へ遷移

### ナビゲーション
- **案件一覧**: 全案件の一覧ページへ移動
- **APIドキュメント**: API仕様書の確認（開発者向け）

## 案件のレビュー

### レビューの流れ

```
買側が案件を登録
  ↓
[新規] ステータス
  ↓
管理者から売側にレビュー依頼
  ↓
売側が内容を確認（ここから）
  ↓
[レビュー中] に変更
  ↓
情報が不足している場合
  → 買側に追加情報を依頼
  → メッセージで質問
  ↓
情報が十分な場合
  → 回答を作成
  → [回答済み] に変更
  ↓
買側が回答内容を確認
  ↓
交渉・条件調整
  ↓
[終了] (成約 or 不成立)
```

### 新規案件の確認手順

#### ステップ1: ダッシュボードで「進行中」案件を確認
- KPIカードの「進行中」数値をチェック
- 最近の案件リストで「新規」バッジを確認

#### ステップ2: 案件詳細を開く
- 案件をクリックして詳細画面へ遷移

#### ステップ3: 買側の要望内容を確認

**確認すべきポイント**:
- [ ] 所在地が正確か（住所、最寄り駅）
- [ ] 土地面積が記載されているか
- [ ] 用途地域・建蔽率・容積率が記載されているか
- [ ] 希望価格が記載されているか
- [ ] 買側の開発計画（200戸マンション）が理解できるか
- [ ] 引渡し時期の希望があるか
- [ ] 備考欄に特記事項があるか

#### ステップ4: 不足情報の確認

**情報が不足している場合**:
- メッセージで買側に追加情報を依頼
- 必要な書類を明示（例: 測量図、登記簿謄本等）
- 回答期限を設定（合理的な期間を設定）

**情報が十分な場合**:
- 回答作成へ進む（次のセクション参照）

## 回答の作成

### 回答作成の手順（API使用）

現在、案件の編集・回答作成はAPI経由で実行します。

#### ステップ1: ログインしてJWTトークンを取得

```bash
curl -X POST https://real-estate-200units-v2.pages.dev/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "seller@example.com",
    "password": "YOUR_PASSWORD"
  }' > token.json

# トークンを環境変数に設定
export TOKEN=$(cat token.json | jq -r '.token')
```

#### ステップ2: 案件を更新して回答を作成

```bash
curl -X PUT https://real-estate-200units-v2.pages.dev/api/deals/{deal_id} \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "status": "REPLIED",
    "remarks": "ご問い合わせありがとうございます。物件の詳細について以下の通り回答いたします。\n\n【土地情報】\n- 測量図添付済み\n- 登記簿謄本添付済み\n- 現況: 更地、即引渡し可能\n\n【価格・条件】\n- 販売価格: 50億円（税込）\n- 引渡し時期: 契約後3ヶ月以内\n- 支払い条件: 契約時30%、引渡し時70%\n\n【その他】\n- 200戸規模のマンション建設に最適な立地です\n- 周辺環境良好、商業施設・学校近接\n- ご不明点があればお気軽にお問い合わせください。"
  }'
```

### 回答内容のポイント

**明確で具体的な情報を提供**:
- 買側の質問に対して、具体的な数値・条件を記載
- 曖昧な表現を避ける（例: 「早め」→「契約後3ヶ月以内」）

**必要な書類を添付**:
- 登記簿謄本
- 測量図
- 現況写真
- その他関連書類

**買側の立場で考える**:
- 買側が意思決定に必要な情報は何か？
- 200戸マンション建設に適した土地であることを示す
- リスク・制限事項も正直に記載（後のトラブル防止）

**次のステップを提案**:
- 現地確認の日程提案
- 追加質問の受付
- 条件交渉の窓口を明示

## メッセージング

### メッセージの送信（API使用）

```bash
# ログイン後、JWTトークンを取得（前述の方法）
export TOKEN=$(cat token.json | jq -r '.token')

# メッセージを送信
curl -X POST https://real-estate-200units-v2.pages.dev/api/deals/{deal_id}/messages \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "content": "物件の測量図を添付いたしました。ご確認いただけますでしょうか？"
  }'
```

### メッセージのベストプラクティス

**迅速な対応**:
- 買側からの質問には24時間以内に返信（営業日ベース）
- 回答に時間がかかる場合は、その旨を伝える

**明確な回答**:
- ❌ 「確認します」
- ✅ 「明日までに測量図を添付してお送りします」

**丁寧な言葉遣い**:
- ビジネス案件なので、常に丁寧な言葉遣いを心がける
- 買側との良好な関係が、円滑な取引につながる

**追加情報の積極的な提供**:
- 買側が聞いていない情報でも、役立つ情報は積極的に提供
- 例: 「なお、近隣に○○がございます」

## ファイル管理

### ファイルのアップロード（API使用）

```bash
# ログイン後、JWTトークンを取得（前述の方法）
export TOKEN=$(cat token.json | jq -r '.token')

# ファイルをアップロード
curl -X POST https://real-estate-200units-v2.pages.dev/api/deals/{deal_id}/files \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/your/file.pdf" \
  -F "file_type=REGISTRY"
```

### ファイルタイプ

| タイプ | 説明 | 例 |
|--------|------|-----|
| OVERVIEW | 物件概要書 | 物件概要.pdf |
| REGISTRY | 登記簿謄本 | 登記簿.pdf |
| MAP | 測量図・地図 | 測量図.pdf |
| REPORT | レポート・調査書 | 地盤調査報告書.pdf |
| OTHER | その他 | 契約書.docx |

### 必須アップロードファイル

売側として、以下のファイルのアップロードを推奨します:

**最優先**:
- [ ] 登記簿謄本（最新のもの）
- [ ] 測量図（境界確定済み）
- [ ] 現況写真（複数枚）

**推奨**:
- [ ] 物件概要書
- [ ] 地盤調査報告書
- [ ] 建築計画概要書（既存建物がある場合）
- [ ] 土地利用計画図
- [ ] 周辺環境マップ

**その他**:
- [ ] 固定資産税評価証明書
- [ ] 道路境界確認書
- [ ] インフラ（電気・水道・ガス）配管図

### ファイルのダウンロード（API使用）

```bash
# ファイル一覧を取得
curl https://real-estate-200units-v2.pages.dev/api/deals/{deal_id}/files \
  -H "Authorization: Bearer $TOKEN"

# 特定のファイルをダウンロード
curl https://real-estate-200units-v2.pages.dev/api/files/{file_id} \
  -H "Authorization: Bearer $TOKEN" \
  -o downloaded_file.pdf
```

## トラブルシューティング

### よくある問題

#### 1. ログインできない

**原因と対処**:
- **パスワードが間違っている**
  → パスワードを再確認してください
  → 管理者にパスワードリセットを依頼してください
- **アカウントが無効化されている**
  → 管理者に連絡して、アカウントの有効化を依頼してください
- **メールアドレスが間違っている**
  → 正しいメールアドレスを確認してください

#### 2. 案件が表示されない

**原因と対処**:
- **フィルターが適用されている**
  → リセットボタンでフィルターをクリア
- **自社に割り当てられた案件のみ表示**
  → 売側ユーザーは自社に割り当てられた案件のみ閲覧可能
- **データベース接続エラー**
  → ページをリロードしてください
  → 解決しない場合は管理者に連絡

#### 3. ファイルがアップロードできない

**原因と対処**:
- **ファイルサイズが大きすぎる**
  → ファイルサイズの上限: 10MB（設定により変更可能）
  → ファイルを圧縮するか、分割してアップロードしてください
- **サポートされていないファイル形式**
  → 対応形式: PDF, Word, Excel, JPEG, PNG
  → 他の形式の場合は、PDFに変換してアップロードしてください

#### 4. 買側から追加質問が来ない

**原因と対処**:
- **買側が回答内容を確認中**
  → ステータスが「回答済み」の場合、買側が確認中です
  → 通常2-3営業日程度かかります
- **回答内容が不十分だった**
  → 買側が意思決定に必要な情報が不足している可能性があります
  → 管理者に連絡して状況を確認してください
- **買側が興味を失った**
  → 条件が合わなかった可能性があります
  → 管理者に連絡して状況を確認してください

#### 5. 通知が届かない

**原因と対処**:
- **通知設定が無効**
  → 設定画面で通知設定を確認してください
  → 必要な通知をONにしてください
- **メールアドレスが間違っている**
  → ユーザー情報でメールアドレスを確認してください
- **迷惑メールフォルダに入っている**
  → 迷惑メールフォルダを確認してください
  → 送信元をホワイトリストに登録してください

### よくある質問（FAQ）

**Q1: 買側から問い合わせが来た後、どのくらいで回答する必要がありますか？**
A1: 通常3-5営業日以内を推奨します。ただし、買側から回答期限が設定されている場合は、その期限を優先してください。

**Q2: 回答後、買側から連絡が来ない場合はどうすればよいですか？**
A2: 通常2-3営業日程度で買側から反応があります。それ以上経過しても連絡がない場合は、管理者に状況を確認してください。

**Q3: 価格交渉は可能ですか？**
A3: はい、可能です。メッセージ機能を使って、買側と条件交渉を進めてください。

**Q4: 買側と直接電話で話すことはできますか？**
A4: システム外での連絡は推奨しません。全てのコミュニケーションはシステム内のメッセージ機能を使用してください。履歴が残り、トラブル防止につながります。

**Q5: 成約した場合、どのような手続きが必要ですか？**
A5: 管理者に連絡し、案件を「終了（成約）」ステータスに変更してもらってください。その後、別途契約手続きを進めてください。

**Q6: 複数の買側から同時に問い合わせがあった場合、どうすればよいですか？**
A6: 先着順または条件の良い買側を優先してください。他の買側には、既に交渉中である旨を伝えてください。
