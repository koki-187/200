## 買側ユーザーの役割

### あなたの役割
買側ユーザーは、**土地の購入を希望する側**として、案件を登録し、売側からの回答を受け取ります。

**主な業務**:
- ✅ 購入希望物件の案件登録
- ✅ 物件情報の詳細入力
- ✅ 売側からの回答確認
- ✅ 質問・追加情報の提供
- ✅ 条件交渉・成約判断
- ✅ ファイル（契約書、レポート等）のアップロード・確認

**権限**:
- ✅ 自社が登録した案件の閲覧・編集
- ✅ 案件へのメッセージ送信
- ✅ ファイルのアップロード・ダウンロード
- ❌ 他社の案件は閲覧不可
- ❌ ユーザー管理・システム設定は不可

## ログイン方法

### ステップ1: ログインページにアクセス
```
URL: https://real-estate-200units-v2.pages.dev/
```

### ステップ2: 買側アカウントでログイン
- **メールアドレス**: 管理者から提供されたメールアドレス
- **パスワード**: 初回ログイン時に設定したパスワード
- **ロール**: AGENT（買側として案件登録）

### ステップ3: ダッシュボードへ自動遷移
ログイン成功後、自動的にダッシュボードに遷移します。

### セキュリティのベストプラクティス
- 強力なパスワードを使用（12文字以上、英数字記号混在）
- パスワードを他人と共有しない
- 定期的にパスワードを変更（3ヶ月ごと推奨）
- 共用PCでは必ずログアウト

## ダッシュボード

### 概要
ダッシュボードでは、あなたが登録した案件の状況を一覧で確認できます。

### KPIカード

#### 1. 総案件数
- 表示内容: あなたが登録した全案件の総数
- 用途: 自社の案件ボリュームを把握

#### 2. 進行中
- 表示内容: ステータスが「新規」「レビュー中」の案件数
- 用途: 売側の回答待ち案件を把握
- アクション: この数値が多い場合、追加情報の提供が必要な可能性

#### 3. 回答済み
- 表示内容: ステータスが「回答済み」の案件数
- 用途: 売側から回答があった案件を確認
- アクション: **重要**: 回答内容を確認し、次のステップへ進める

#### 4. 完了
- 表示内容: ステータスが「終了」の案件数
- 用途: 成約または不成立で終了した案件数
- アクション: 成約案件は契約手続きへ進む

### 最近の案件
- 表示件数: 最新5件
- 表示情報: 案件名、ステータス、所在地、作成日
- アクション: クリックで案件詳細へ遷移

### ナビゲーション
- **案件一覧**: 全案件の一覧ページへ移動
- **新規案件登録**: 新しい案件を作成
- **APIドキュメント**: API仕様書の確認（開発者向け）

## 案件の新規登録

### 新規案件登録の流れ

```
買側が案件を登録
  ↓
[新規] ステータス
  ↓
管理者が内容を確認
  ↓
売側にレビュー依頼
  ↓
[レビュー中] に変更
  ↓
売側が回答作成
  ↓
[回答済み] に変更
  ↓
買側が回答内容を確認
  ↓
交渉・条件調整
  ↓
[終了] (成約 or 不成立)
```

### 登録手順（API使用）

現在、案件登録はAPI経由で実行します。

#### ステップ1: ログインしてJWTトークンを取得

```bash
curl -X POST https://real-estate-200units-v2.pages.dev/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "buyer@example.com",
    "password": "YOUR_PASSWORD"
  }' > token.json

# トークンを環境変数に設定
export TOKEN=$(cat token.json | jq -r '.token')
```

#### ステップ2: 新規案件を登録

```bash
curl -X POST https://real-estate-200units-v2.pages.dev/api/deals \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "title": "渋谷区代々木200戸用地",
    "location": "東京都渋谷区代々木1-1-1",
    "station": "代々木駅",
    "walk_minutes": 5,
    "land_area": "2000㎡",
    "zoning": "商業地域",
    "building_coverage": "80%",
    "floor_area_ratio": "600%",
    "height_district": "高度利用地区",
    "fire_zone": "防火地域",
    "road_info": "南側6m公道、東側4m公道",
    "current_status": "更地",
    "desired_price": "50億円",
    "remarks": "早期売却希望、200戸規模のマンション用地として最適",
    "seller_id": "SELLER_USER_ID"
  }'
```

### 入力項目の説明

#### 必須項目
| 項目 | 説明 | 例 |
|------|------|-----|
| title | 案件名（わかりやすいタイトル） | 渋谷区代々木200戸用地 |
| location | 所在地（住所） | 東京都渋谷区代々木1-1-1 |
| seller_id | 売側ユーザーID（管理者から提供） | xxx-xxx-xxx |

#### 推奨項目
| 項目 | 説明 | 例 |
|------|------|-----|
| station | 最寄り駅 | 代々木駅 |
| walk_minutes | 駅徒歩（分） | 5 |
| land_area | 土地面積 | 2000㎡ |
| zoning | 用途地域 | 商業地域 |
| building_coverage | 建蔽率 | 80% |
| floor_area_ratio | 容積率 | 600% |
| height_district | 高度地区 | 高度利用地区 |
| fire_zone | 防火地域 | 防火地域 |
| road_info | 道路情報 | 南側6m公道 |
| current_status | 現況 | 更地 |
| desired_price | 希望価格 | 50億円 |
| remarks | 備考（その他情報） | 早期売却希望 |

### 案件登録時のポイント

**詳細な情報を入力する**:
- 情報が充実しているほど、売側が適切な回答を提供できます
- 不足情報があると、売側から追加情報の依頼が来る可能性があります

**正確な情報を入力する**:
- 誤った情報は後のトラブルの原因となります
- 不明な項目は「備考」に「要確認」と記載してください

**希望条件を明確にする**:
- 希望価格、引渡し時期、支払い条件など
- 買側の開発計画（200戸マンションの概要）を共有すると、売側の理解が深まります

## 売側からの回答確認

### 回答済み案件の確認手順

#### ステップ1: ダッシュボードで「回答済み」案件を確認
- KPIカードの「回答済み」数値をチェック
- 最近の案件リストで「回答済み」バッジを確認

#### ステップ2: 案件詳細を開く
- 案件をクリックして詳細画面へ遷移

#### ステップ3: 売側の回答内容を確認
- **基本情報タブ**: 売側が更新した物件情報
- **ファイルタブ**: 売側がアップロードした書類（登記簿、測量図等）
- **メッセージタブ**: 売側からのコメント・回答

#### ステップ4: 回答内容の評価

**確認すべきポイント**:
- [ ] 希望条件が満たされているか
- [ ] 価格が予算内か
- [ ] 必要な書類が揃っているか
- [ ] 法的制限が開発計画に影響しないか
- [ ] 引渡し時期が希望と合致しているか

**次のアクション**:
- **興味がある場合**: メッセージで追加質問・条件交渉
- **条件が合わない場合**: 管理者に連絡して終了を依頼
- **判断保留の場合**: 追加情報を依頼

## メッセージング

### メッセージの送信（API使用）

現在、メッセージ送信はAPI経由で実行します。

```bash
# ログイン後、JWTトークンを取得（前述の方法）
export TOKEN=$(cat token.json | jq -r '.token')

# メッセージを送信
curl -X POST https://real-estate-200units-v2.pages.dev/api/deals/{deal_id}/messages \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "content": "物件について追加で質問があります。建物の現況について詳しく教えていただけますか？"
  }'
```

### メッセージのベストプラクティス

**明確な質問をする**:
- ❌ 「詳しく教えてください」
- ✅ 「建物の築年数と現在の使用状況を教えてください」

**具体的な情報を提供する**:
- ❌ 「早めに回答してください」
- ✅ 「弊社の取締役会が来週水曜日のため、それまでに回答いただけると助かります」

**丁寧な言葉遣い**:
- ビジネス案件なので、常に丁寧な言葉遣いを心がけてください
- 売側との良好な関係が、円滑な取引につながります

## ファイル管理

### ファイルのアップロード（API使用）

現在、ファイルアップロードはAPI経由で実行します。

```bash
# ログイン後、JWTトークンを取得（前述の方法）
export TOKEN=$(cat token.json | jq -r '.token')

# ファイルをアップロード
curl -X POST https://real-estate-200units-v2.pages.dev/api/deals/{deal_id}/files \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/your/file.pdf" \
  -F "file_type=REPORT"
```

### ファイルタイプ

| タイプ | 説明 | 例 |
|--------|------|-----|
| OVERVIEW | 物件概要書 | 物件概要.pdf |
| REGISTRY | 登記簿謄本 | 登記簿.pdf |
| MAP | 測量図・地図 | 測量図.pdf |
| REPORT | レポート・調査書 | 地盤調査報告書.pdf |
| OTHER | その他 | 契約書.docx |

### ファイルのダウンロード（API使用）

```bash
# ファイル一覧を取得
curl https://real-estate-200units-v2.pages.dev/api/deals/{deal_id}/files \
  -H "Authorization: Bearer $TOKEN"

# 特定のファイルをダウンロード
curl https://real-estate-200units-v2.pages.dev/api/files/{file_id} \
  -H "Authorization: Bearer $TOKEN" \
  -o downloaded_file.pdf
```

### OCR機能の利用

画像ファイル（PDF、JPEG、PNG）をアップロードすると、自動的にテキストが抽出され、案件情報に反映されます。

**対応ファイル形式**:
- PDF
- JPEG / JPG
- PNG

**利用シーン**:
- 紙の登記簿謄本をスキャンしてアップロード
- 手書きの測量図を画像化してアップロード
- 現地調査メモの写真をアップロード

## トラブルシューティング

### よくある問題

#### 1. ログインできない

**原因と対処**:
- **パスワードが間違っている**
  → パスワードを再確認してください
  → 管理者にパスワードリセットを依頼してください
- **アカウントが無効化されている**
  → 管理者に連絡して、アカウントの有効化を依頼してください
- **メールアドレスが間違っている**
  → 正しいメールアドレスを確認してください

#### 2. 案件が表示されない

**原因と対処**:
- **フィルターが適用されている**
  → リセットボタンでフィルターをクリア
- **他社の案件を見ようとしている**
  → 買側ユーザーは自社が登録した案件のみ閲覧可能
- **データベース接続エラー**
  → ページをリロードしてください
  → 解決しない場合は管理者に連絡

#### 3. ファイルがアップロードできない

**原因と対処**:
- **ファイルサイズが大きすぎる**
  → ファイルサイズの上限: 10MB（設定により変更可能）
  → ファイルを圧縮するか、分割してアップロードしてください
- **サポートされていないファイル形式**
  → 対応形式: PDF, Word, Excel, JPEG, PNG
  → 他の形式の場合は、PDFに変換してアップロードしてください

#### 4. 売側から回答が来ない

**原因と対処**:
- **売側がまだレビュー中**
  → ステータスが「レビュー中」の場合、売側がまだ回答を作成中です
  → 通常3-5営業日程度かかります
- **追加情報が必要**
  → メッセージタブで売側からの質問がないか確認してください
  → 不足情報がある場合は、速やかに提供してください
- **回答期限を過ぎている**
  → 管理者に連絡して状況を確認してください

#### 5. 通知が届かない

**原因と対処**:
- **通知設定が無効**
  → 設定画面で通知設定を確認してください
  → 必要な通知をONにしてください
- **メールアドレスが間違っている**
  → ユーザー情報でメールアドレスを確認してください
- **迷惑メールフォルダに入っている**
  → 迷惑メールフォルダを確認してください
  → 送信元をホワイトリストに登録してください

### よくある質問（FAQ）

**Q1: 案件を登録した後、どのくらいで回答がもらえますか？**
A1: 通常3-5営業日程度です。売側がレビューを完了すると、ステータスが「回答済み」に変わります。

**Q2: 複数の売側に同時に問い合わせできますか？**
A2: はい、可能です。案件ごとに異なる売側ユーザーIDを指定して、複数の案件を登録してください。

**Q3: 一度登録した案件の情報を修正できますか？**
A3: はい、可能です。案件詳細画面で「編集」ボタンをクリックして修正してください。

**Q4: 売側と直接電話で話すことはできますか？**
A4: システム外での連絡は推奨しません。全てのコミュニケーションはシステム内のメッセージ機能を使用してください。履歴が残り、トラブル防止につながります。

**Q5: 成約した場合、どのような手続きが必要ですか？**
A5: 管理者に連絡し、案件を「終了（成約）」ステータスに変更してもらってください。その後、別途契約手続きを進めてください。
