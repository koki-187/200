## システム概要

**200棟土地仕入れ管理システム** は、不動産仲介業者向けに開発された土地仕入れ案件の一元管理システムです。本システムは以下の3つのステークホルダーの視点から設計されています：

### 主要ステークホルダー
- **買主（投資家）**: 投資判断に必要な情報を正確かつ迅速に取得
- **買側仲介業者**: OCRによる効率的な物件情報入力と案件管理
- **売側仲介業者**: リアルタイムな案件状況の可視化と迅速な対応

### 主要機能
- **OCR自動入力**: 登記簿謄本や物件資料から17項目を自動抽出（iOS Safari完全対応）
- **物件情報自動取得**: 国土交通省APIから用途地域・建ぺい率等を自動取得
- **案件管理**: 52のコア機能による包括的な案件ライフサイクル管理
- **ユーザー管理**: ADMIN、買側AGENT、売側AGENTの3ロール管理
- **PWA対応**: オフライン機能とホーム画面への追加をサポート

### 本番環境URL
- **最新版**: https://real-estate-200units-v2.pages.dev/
- **最新デプロイ**: https://b3bb6080.real-estate-200units-v2.pages.dev/

## 案件管理

### 新規案件の登録

#### 手順
1. ダッシュボードまたはヘッダーメニューから「新規案件」をクリック
2. 案件登録フォーム（`/deals/new`）に遷移
3. 以下の方法で情報を入力：

##### 方法1: OCR自動入力（推奨）
1. 「ファイルを選択」ボタンをクリック
2. 登記簿謄本や物件資料の写真/PDFをアップロード
3. OCRが自動的に17項目を抽出して入力：
   - 物件名、住所、土地面積、建物面積、間口
   - 用途地域、建ぺい率、容積率、構造、築年月
   - 希望売出価格、表面利回り、賃貸状況など
4. 自動入力された内容を確認・修正

##### 方法2: 物件情報自動取得
1. 住所を入力（例：`東京都港区六本木1-1-1`）
2. 「物件情報を取得」ボタンをクリック
3. 国土交通省APIから以下の情報を自動取得：
   - 用途地域
   - 建ぺい率
   - 容積率
   - 前面道路情報
   - 最寄駅距離
4. 取得された内容を確認・修正

##### 方法3: 手動入力
各フィールドに直接情報を入力

#### 必須フィールド（17項目）
- **基本情報**: 物件名、住所
- **土地情報**: 土地面積、間口、用途地域、建ぺい率、容積率
- **建物情報**: 建物面積、構造、築年月
- **金融情報**: 希望売出価格、表面利回り
- **状況**: 賃貸状況、備考

#### 保存
1. 全ての必須項目を入力
2. 「保存」ボタンをクリック
3. 案件詳細ページに遷移

## OCR機能の使い方

### 対応デバイス
- **iOS Safari**: 完全対応（v3.108.0で完全修正）
- **Android Chrome**: 対応
- **デスクトップブラウザ**: 対応

### サポートされているファイル形式
- **画像**: JPEG, PNG, HEIC（iOS）
- **ドキュメント**: PDF

### OCR実行手順（iOS Safari）

#### ステップ1: ファイル選択
1. `/deals/new` ページで「ファイルを選択」ボタンをタップ
2. iOSのファイル選択画面が表示される
3. 以下の選択肢から選択：
   - **写真ライブラリ**: 既存の写真を選択
   - **書類をスキャン**: カメラで書類を撮影
   - **ファイル**: iCloud DriveやFiles appから選択

#### ステップ2: OCR処理
1. ファイルを選択すると自動的にOCR処理が開始
2. 処理中は「OCR処理中...」と表示（約10〜30秒）
3. 処理完了後、自動的に17項目がフォーム入力される

#### ステップ3: 確認と修正
1. 自動入力された内容を確認
2. 必要に応じて手動で修正
3. 「保存」ボタンをクリック

### OCRで抽出される情報（17項目）
1. **物件名** (`property_name`)
2. **住所** (`address`)
3. **土地面積** (`land_area`) - ㎡単位
4. **建物面積** (`building_area`) - ㎡単位
5. **間口** (`frontage`) - m単位
6. **用途地域** (`zoning`)
7. **建ぺい率** (`building_coverage`) - %
8. **容積率** (`floor_area_ratio`) - %
9. **構造** (`structure`) - 例：木造、RC造
10. **築年月** (`built_year`) - YYYY-MM形式
11. **希望売出価格** (`desired_price`) - 万円単位
12. **表面利回り** (`yield`) - %
13. **賃貸状況** (`occupancy`) - 例：満室、空室
14. **最寄駅** (`nearest_station`)
15. **駅距離** (`station_distance`) - 分
16. **前面道路** (`road_info`)
17. **備考** (`remarks`)

### OCRのトラブルシューティング

#### 問題1: OCRが開始しない
**原因**: ファイル選択のchangeイベントが発火していない
**解決策**:
- ページをリロード（Shift + F5）
- 別のブラウザで試行
- iOS Safariの場合、設定 > Safari > 詳細 > JavaScriptをオンに確認

#### 問題2: OCR処理が途中で止まる
**原因**: ネットワークタイムアウトまたはAPI制限
**解決策**:
- 30秒待機（iOS版は30秒タイムアウト設定）
- Wi-Fi接続を確認
- 画像サイズを縮小（推奨: 5MB以下）

#### 問題3: 抽出精度が低い
**原因**: 画像の品質が低い
**解決策**:
- 明るい場所で撮影
- 手ブレを防ぐ
- 書類全体が写るように撮影
- PDFの場合、テキスト埋め込み型を使用

## 物件情報自動取得

### 国土交通省 不動産情報ライブラリ API

#### 機能概要
住所を入力するだけで、国土交通省の公開データから物件情報を自動取得できます。

#### 使用方法
1. `/deals/new` ページで住所を入力
   - 例：`東京都港区六本木1-1-1`
2. 「物件情報を取得」ボタンをクリック
3. APIから以下の情報を自動取得：
   - **用途地域**
   - **建ぺい率**
   - **容積率**
   - **前面道路情報**
   - **最寄駅距離**
4. 取得された情報が自動的にフォームに入力される

#### 注意事項
- **取引時期**: 最新の四半期データを使用（例：2024年第4四半期）
- **データ件数**: 1回のリクエストで最大232件のデータを取得
- **精度**: 住所の正確性により結果が変動
- **API制限**: 1日あたりのリクエスト数に制限あり（詳細は管理者に確認）

#### トラブルシューティング
- **データが取得できない場合**: 住所を詳細に入力（番地まで）
- **古いデータしか表示されない場合**: 最新の取引がない地域の可能性
- **APIエラーが発生する場合**: しばらく時間をおいて再試行

## ファイル管理

### ストレージクォータ
- **ユーザーあたりの容量**: 3GB（3,072MB）
- **最大ファイルサイズ**: 10MB/ファイル
- **対応ファイル形式**: 
  - 画像: JPEG, PNG, HEIC
  - ドキュメント: PDF
  - その他: Excel, Word, PowerPoint

### ファイルのアップロード
1. 案件詳細ページで「ファイル」タブをクリック
2. 「ファイルをアップロード」ボタンをクリック
3. ファイルを選択（複数選択可能）
4. アップロード完了後、ファイル一覧に表示

### ファイルのダウンロード
1. ファイル一覧から対象ファイルをクリック
2. ダウンロードが自動的に開始

### ファイルの削除
1. ファイル一覧から「削除」ボタンをクリック
2. 確認ダイアログで「OK」をクリック

### ストレージ使用量の確認
1. ダッシュボードまたは設定ページで確認可能
2. 表示項目：
   - 使用済み容量
   - 残り容量
   - ファイル数（OCR用、写真、その他）

### よくある質問（FAQ）

3. ログがない場合、ページをリロード
4. それでも解決しない場合、管理者に連絡

3. ブラウザコンソールでエラーを確認
4. それでも解決しない場合、管理者に連絡

#### Q4: favicon.icoが404エラー
**A**: v3.151.0で完全修正されました：
1. ブラウザキャッシュをクリア（Shift + F5）
2. 最新デプロイURL（https://b3bb6080.real-estate-200units-v2.pages.dev/）で確認
3. それでも表示されない場合、管理者に連絡

#### Q5: ファイルがアップロードできません
**A**: 以下を確認してください：
1. ファイルサイズが10MB以下か
2. ストレージクォータが残っているか（3GB制限）
3. ファイル形式が対応しているか
4. ネットワーク接続が安定しているか

#### Q6: 案件が保存できません
**A**: 以下を確認してください：
1. 必須フィールド（17項目）が全て入力されているか
2. 数値フィールドに正しい形式で入力されているか
3. ブラウザコンソールでエラーを確認
4. ネットワーク接続が安定しているか

### エラーコードとその対処法

#### ERR_OCR_TIMEOUT
**原因**: OCR処理が30秒以内に完了しなかった
**対処法**:
1. 画像サイズを縮小（5MB以下推奨）
2. Wi-Fi接続を確認
3. 再度アップロード

#### ERR_API_AUTH_FAILED
**原因**: 認証トークンが無効または期限切れ
**対処法**:
1. ログアウト後、再ログイン
2. ブラウザのキャッシュをクリア
3. それでも解決しない場合、管理者に連絡

#### ERR_STORAGE_QUOTA_EXCEEDED
**原因**: ストレージクォータ（3GB）を超過
**対処法**:
1. 不要なファイルを削除
2. 管理者にクォータ増量を依頼

#### ERR_FILE_TOO_LARGE
**原因**: ファイルサイズが10MBを超過
**対処法**:
1. ファイルサイズを圧縮
2. 複数のファイルに分割
